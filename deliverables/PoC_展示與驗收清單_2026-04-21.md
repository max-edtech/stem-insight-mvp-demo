# PoC 展示與驗收清單（父親／中小建商版）

日期：2026-04-21  
定位：不修改甘特圖與合作報價圖，直接用系統功能做展示與驗收。

## 0. 驗收目標（先講結論）
本次驗收只看 3 件事：
1. 拿地前能否快速判斷「可不可做、保本線在哪」。  
2. 施工中能否把「變更、預算、請回款」串成同一個控利流程。  
3. 老闆能否在同一畫面看到「稅前/稅後獲利與風險」。

---

## 1. 展示路線（45-60 分鐘）

### A 段：前期可行性（10-15 分）
- 頁面：`/new-project`
- 展示動作：
  1. 輸入土地、營造、融資、售價參數。
  2. 即時觀察：可售面積、損益兩平售價、獲利率、ROE、淨利。
  3. 建立案件並跳轉到 `/{projectId}`。
- 驗收標準：
  - 參數改動 3 秒內看到結果變化。
  - 可明確說出保本售價與目標獲利率差距。

### B 段：中期控利（15-20 分）
- 頁面：`/{projectId}`、`/{projectId}/compare`、`/{projectId}/operations`
- 展示動作：
  1. 在總覽看主項預算分布與 EAC 預估。
  2. 進 `operations` 新增 1 筆變更單（狀態 `approved`）。
  3. 回總覽確認「現行預算」已回寫。
  4. 到 `compare` 確認差異有反映。
- 驗收標準：
  - 核准變更後，預算總額立即更新。
  - 差異分析頁可看到主項差異方向與金額。

### C 段：現金流與稅後（15-20 分）
- 頁面：`/{projectId}/operations`
- 展示動作：
  1. 建 1 筆請款（billing）。
  2. 建 1 筆回款（collection）。
  3. 檢查請款缺口是否正確（請款 - 回款）。
  4. 建 1 組稅務情境（課稅所得＋稅率）。
  5. 檢查稅後獲利是否更新。
- 驗收標準：
  - 請回款新增後，缺口卡片同步更新。
  - 稅務情境儲存後，可看到稅後獲利與稅務紀錄。

### D 段：決策輸出（5 分）
- 頁面：`/api/projects/{projectId}/export`
- 展示動作：下載 Excel 報表。
- 驗收標準：
  - 檔案可下載且至少包含：`專案摘要`、`預算表`、`A案計價`、`進度表`、`現金流`、`變更單`、`稅務情境`、`主項彙總`、`預算比對`、`EAC預測`（有資料時）。

---

## 2. 驗收矩陣（當場打勾）

| 項目 | 驗收問題 | 通過條件 | 結果 |
|---|---|---|---|
| 前期試算 | 是否可快速得出保本線與獲利率？ | 可即時重算且數字一致 | ☐ Pass / ☐ Fail |
| 變更回寫 | 核准變更是否回寫預算？ | `operations` 與總覽預算一致 | ☐ Pass / ☐ Fail |
| 差異分析 | 能否定位哪個主項偏差？ | `compare` 可顯示主項差異金額 | ☐ Pass / ☐ Fail |
| 請回款 | 請款缺口是否可追蹤？ | 缺口 = 請款 - 回款 | ☐ Pass / ☐ Fail |
| 稅務情境 | 是否可看到稅後獲利？ | 可新增情境並更新稅後獲利 | ☐ Pass / ☐ Fail |
| 報表輸出 | 會議後能否交付檔案？ | Excel 成功下載且分頁完整 | ☐ Pass / ☐ Fail |

---

## 3. 最小可驗收資料（會前準備）

1. 一組可講清楚的基本參數（地坪、容積、營造單價、售價、利率）。  
2. 一筆示範變更單（編號、標題、金額、核准狀態）。  
3. 一筆請款與一筆回款（項目、日期、金額）。  
4. 一組稅務假設（課稅所得、稅率）。  

註：若要把 EAC 從「示範」提升到「管理級」，會後再補 10 筆以上非 0 實支資料。

---

## 4. 當場結案判定（Go / Revise）

- **Go（可進下一案）**：驗收矩陣 6 項中至少 5 項 Pass，且「變更回寫、請回款、稅後獲利」三項必須全過。  
- **Revise（需修正）**：任一核心項目 Fail，48 小時內補修並安排二次驗收。
